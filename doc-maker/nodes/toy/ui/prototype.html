使用说明 ToyNode Console · 作者活体地图

这个 console 是给作者本人用的——让你几周不碰也能立刻拾回这个节点的状态。不是协作 UI、不是新人 onboarding 工具。详见 docs/05-ui-spec.md §0。

日常 4 动作循环

  • 1跑批：终端跑 s2v run toy --cases ...
  • 2审产物：看「评估归因」和「决策追溯」→ 字段旁 [💬] 提反馈
  • 3改物料：按 L0→L5 升级路径，先一次过率分诊（详见 04-handbook §4.1）
  • 4验证：s2v eval regression 双向 Gate（train + holdout 都看）

主要按钮

  • ⟳ 重跑：UI 仅作壳，内部调 s2v run CLI
  • 💬 反馈：填结构化表单，落 nodes/toy/feedback/<id>.yaml 入 git
  • 物料徽章：点击看版本和内容；编辑必经 git（UI 不在浏览器写物料文件）
  • 📝 在 $EDITOR 中编辑：触发本地编辑器；改完物料必走 Promote Pipeline
  • ▶ 查看完整 rendered prompt：决策追溯卡片底部 —— 看最终发给 LLM 的字符串

三条铁律 写在墙上 (04-handbook §9)

① 单条反馈不立即改 prompt。≥ 3 条同 likely_cause 才触发改物料（不变量 #15）。单条 = 噪声。
② 物料 bump 必须 train 改进 + holdout 不退化双满足。单向 = 过拟合（不变量 #14）。
③ 撞 K=3 轮无改进 → 强制升级 L0→L5。死磕 = 假装在工作，业务永远等不到（不变量 #16）。
