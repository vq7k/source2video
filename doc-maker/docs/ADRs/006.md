# ADR-006 · 模型分档 + Judge 策略

**Status**: Accepted

**Decision**
三档够用：

| 档位 | 用途 |
|---|---|
| 顶档（Opus / GPT-5 同级） | 内容规划、judge、复杂图表理解 |
| 中档（Sonnet 同级） | 脚本生成主力、大部分 agent 节点 |
| 小档（Haiku 同级 / 微调） | 高量重复转换、格式化、初筛 |

**Judge 规则**
- Judge 必须 ≥ generator 同代。
- **同节点 generator 与 judge 可同模型**——跨模型 judging 引入的方差大于客观性收益（除非刻意做 ensemble）。

**Rationale**
- 用弱模型 judge 强模型是常见错误，judge 信号偏移会带偏整个优化方向。
- 跨模型 judging 的"独立性"是幻觉，实际是新的偏置源。
