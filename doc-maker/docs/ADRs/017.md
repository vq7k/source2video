# ADR-017 · 最小物料 = 1 份 exemplar（graceful degrade）

**Status**: Accepted

**Context**
- 早期 Node 启动校验设计太严：要求 5 类物料齐 + 每类都有 anchor / 反例。
- 这违反了"v1.0 bootstrap 例外"（[ADR-008](008.md)）+ [ADR-005](005.md) 阶梯精神（"从下往上爬，够用就停"）。
- 用户问"最小物料"，期望的是一份示例文档，不是 5 类全套。

**Alternatives**
1. 5 类物料齐，每类强约束（早期错误）。
2. 3 类必备（prompts + schemas + rubrics）。
3. **最小 = exemplar + prompt + schema 三件；其它 graceful degrade**（采纳）。

**Decision**
- **硬约束（缺则启动 hard fail）**：仅 prompts（无法 LLM call）+ schemas（无法 validate）两类。
- **graceful degrade**：
  - 缺 rubric → judge 跳过，artifact 仍产出，标 `eval: skipped`
  - 缺 style_guide → 不注入 style 块
  - 缺 exemplars → few-shot 不注入
- **最小启动**：1 份 exemplar 草示意 + 一句话 prompt + 1 个默认 schema 就能跑。

**Rationale**
- bootstrap 阶段过度约束 = 项目根本起不来。
- [ADR-005](005.md) "Pattern 阶梯"明确"从下往上爬"——节点先用最薄起步，撞痛点再补。
- 完整 5 类齐是演化的终态，不是启动门槛。

**Consequences**
- [`../02-architecture.md`](../02-architecture.md) §2.2 最小物料约束改写：硬 fail 仅 prompts + schemas。
- 物料校验区分 "必备" 与 "可省，但若有必须合规"。
- US-12 AC 反映：Console 必须展示所有 5 类物料徽章（即使为空也要显式标 None）。
