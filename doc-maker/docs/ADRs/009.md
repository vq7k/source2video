# ADR-009 · 评审治理：评分表 + 评审校准 + 与下游相关性校验

**Status**: Accepted

**Context**
**人工评审 plan 不可靠且会漂移**。典型漂移：

- **Anchor drift**（标杆漂移）：早期看的几个样本锚定了"好"的标准，后续标准被永久压低或抬高
- **Fatigue drift**（疲劳漂移）：连续评 10 个之后标准会松；早上 vs 晚上分数不一样
- **Recency bias**（近因偏差）：刚看完出彩的，下一个普通看着就差
- **Inter-reviewer variance**（评审者间方差）：两个评审对同一份 plan 经常分歧
- **Goodhart 效应**（指标反噬）：一旦把 rubric 当成目标去优化，rubric 内分数越来越高、rubric 外漂走——你以为质量在涨，实际真实质量在退（出处：经济学家 Charles Goodhart）
- **Mental rubric 漂移**：评审脑里的标准比纸面更细但不稳定，自己也说不清

**最深的问题**：Plan 看起来好 ≠ 视频做出来好。**最危险的不是评审给错分，是评审给对了分但跟下游质量无关**。

**Decision（按优先级落实）**
1. **评分表**（书面化评审标准，不是脑里的）：5–7 维度，每维度配 0/5/10 分 **anchor example（标杆例子：0 分长什么样 / 5 分长什么样 / 10 分长什么样，给评审对照打分用，不靠脑里的标准）** + 反例集。
2. **每月 评审校准**：盲评 10 个旧 plan，看个人漂移 / 组内方差 / 整体漂移。
3. **对照样本**（混入评审池的"已知差"样本）：评审池混 10% 已知差 plan，给高分 = 标准漂了。
4. **多评审 + 一致性**：关键决策最少 2 人盲评，分歧 > 阈值召第三人。
5. **季度相关性校验**（最关键）：plan 评分 vs 真实下游视频质量（用户反馈 / 完播率 / 内部抽检）做相关性。低 → 评审在自嗨，rubric 重做。
6. **长期方向**：plan 评审权重逐步下降，让位给下游成品指标（Stage 1→4）。

**Rationale**
- Plan 评审是拐杖，不是尺子。尺子永远在下游。
- 不做相关性校验 → 3 个月后才发现优化方向跑偏。
