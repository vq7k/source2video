# ADR-020 · UI 为作者活体地图（不是协作工具）

**Status**: Accepted

**Context**
- 早期把 Per-Node Console 当作"协作便利工具"，标 P1 延后。
- 用户原话："**没有 UI 我对项目的理解需要看文档看代码。没有 UI 项目架构和设计随着时间会在我脑子里死去**。"
- 真正的设计动机被误读：UI 不是给新人用的，是给作者本人维持对项目的活体感知用的。

**Alternatives**
1. UI = 协作便利工具（P1 延后）。
2. UI = 新人 onboarding 工具（依然 P1）。
3. **UI = 作者本人的项目活体地图**（采纳，P0）。

**Decision**
- Per-Node Console 升 P0，**必须进 MVP**。
- 不是为新人 / 评审者 / 协作者设计，是为作者本人服务。
- 单页 Streamlit ≤ 200 行，artifact + 物料 + Eval Attribution + Decision Trace + 反馈表单 + rerun + 节点指标六件一屏。

**Rationale**
- 没 console = 节点设计 + 物料 + 当前指标散在文件系统多处，几周不碰认知冷却 → 重启靠考古。
- console = 一屏看完，活体感知。**是认知负担减压器，不是协作便利的糖**。
- 第一版只有一个用户（作者本人），但这个用户最大的痛是"自己对自己项目的认知能否持续"——console 直接解这个。

**Consequences**
- US-12 P1 → **P0**；MVP 出口判据 #7 加 Console 上线。
- [`../06-ui-spec.md`](../06-ui-spec.md) 新建，含 ASCII 原型 + 实施栈 + 反模式。
- 跟 [ADR-021](021.md) "无 Web UI 上传 source"区分：UI 上传 source 不要，**Per-Node review/rerun console 要**。
- [`../02-architecture.md`](../02-architecture.md) §6 MVP 范围六模块齐，Console 不再"不在 MVP 范围"。
