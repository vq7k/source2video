# ADR-018 · Synthetic Fixtures（业务数据不进框架测试）

**Status**: Accepted

**Context**
- 早期 fixtures/cases/ 用 ai-engineer-roadmap 真实 markdown（KNN 章节 README 等）。
- 用户指出："框架测试不应该混入业务"——用业务数据测框架违反 [ADR-011](011.md) 纪律。

**Alternatives**
1. 真实业务数据当 fixtures（早期错误）。
2. **全 synthetic markdown，业务无关**（采纳）。
3. 混合（合成 + 抽样真实）。

**Decision**
- `fixtures/cases/` 全部人造 markdown（合成饮料 / 一行日记 / 数学符号 / 含代码 / Unicode 等 5 个正例 + 3 个 TC-NEG-*）。
- `fixtures/materials/toy/v1.0/` 全部人造 prompt / rubric，跟讲解视频毫无关系。
- 业务数据装载 / 老规则灌入框架 = **第二轮业务侧 MVP 的事**（[`../_future/business-design.md`](../_future/business-design.md) §10a）。

**Rationale**
- 隔离性：框架 bug ≠ 业务 bug，分得清才能 debug。
- 可控性：synthetic 可精确覆盖边界（极短 / 极长 / 含代码 / Unicode）。
- 稳定性：业务数据在动（ai-engineer-roadmap 在跑），synthetic frozen 不变。
- 纪律一致：跟 [ADR-011](011.md) 一脉相承。

**Consequences**
- 删 US-06 旧版本（"真实业务数据装载兼容"），改为"Synthetic 测试用例 + CI 集成"。
- [`../07-acceptance.md`](../07-acceptance.md) §3 完整重写为测试用例设计表 + 内容详述 + 回归基线 + CI 集成。
- TC-01 ~ TC-05 + TC-NEG-01 ~ 03 共 8 个 case 覆盖框架 90% 能力面。
- 真实业务数据装载留给第二轮 [`../_future/business-design.md`](../_future/business-design.md) §10a。
