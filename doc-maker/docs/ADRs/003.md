# ADR-003 · 跨节点协调：分层粒度，shot 之间不互读

**Status**: Accepted

**Context**
- 一期视频（一章，~10 分钟）= 多个 shot，shot 之间要保持风格、上下文、叙事一致。
- 当前是"按期"为事务单元（不可再分），痛点：改 1 个 shot 要重做整期。
- 扁平拆到 shot 会让 shot 之间为了维持一致性互相读 → 退化成串行依赖。

**Decision**
分层粒度：

```
系列级（整门课，可选）
  ├── 系列 plan / 术语表 / 人设
  └── 期级（一章一视频）
        ├── 期级 plan         ← shot list + 时长配比 + 叙事弧 + callback 设计
        ├── 期级 style guide  ← 声音、镜头语言、术语表（只读 const）
        ├── shot 级执行（并行，独立 retry）
        └── 期级一致性校验    ← 衔接 / 风格 drift / 总时长 / 查重
```

**铁律：shot 之间不横向通信**。所有跨 shot 约束 → 预协调到 plan 里，或后置到 QA 节点。

**Rationale**
- shot 看不见兄弟的"输出"，但看得见兄弟的"意图"（通过 plan）。这是分层能成立的唯一基础。
- 横向通信废掉独立重做、并行执行能力，引入雪球失败。
- 缓存失效边界变干净：改单 shot 只重做单 shot；改 plan → 下游全部失效（预期行为）。

**Verification（强制）**
挑一期满意视频 → 人工反推 plan → 判断：给定 plan + style + 原料，shot 能否"在不读兄弟输出的前提下"被独立正确生成。
- 能 → 分层成立。
- 不能 → 通常是 plan 信息密度不够，补字段直到能承载；极少数情况才说明分层在此场景失败。

**Consequences**
- plan artifact 成为系统宪法，质量决定整个 pipeline 的天花板（→ [ADR-004](004.md)）。
- 系列级（多期一致性）按同构 hierarchy 递归套用，**当前不实现**，撞到痛点再说。
