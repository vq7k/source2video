# ADR-026 · UI 实施栈从 Streamlit 改为 Next.js（全栈统一）

**Status**: Accepted（2026-05-13，覆盖 [08-tech-stack §2.5](../08-tech-stack.md) 原 Streamlit 选型）

> 2026-05-15 更新：L1 Business Console 已由 [ADR-027](027.md) 从 `_future/` 提升为当前业务产品原型 SOT，当前文档为 [`../business-console.md`](../business-console.md)。

**Context**

- [08-tech-stack §2.5](../08-tech-stack.md) 原选 Streamlit 实施 UI，理由：单页 ≤ 200 行 / 不学新框架 / per-console 独立进程。
- 此选型的隐含前提：UI 只服务于"作者活体地图"（[ADR-020](020.md)），没有面向使用者的视觉要求。
- **[ADR-025](025.md) 改变了这个前提**——加入 L1 业务层 console（面向使用者，视觉是产品基本盘）+ L2 Hub Console（跨节点导航）+ L3 Per-Node Console。
- Streamlit 适合 L3（作者活体地图），**不适合 L1**（使用者视觉）+ **不适合 L2**（跨页路由）：
  - 视觉天花板低（组件粗糙、动效零、布局只有 hstack/vstack）
  - 状态管理弱（session_state 难维护，rerun 对长任务/轮询是 hack）
  - 路由 / 多页弱（multipage 是 hack 不是设计）
  - 前端生态接不上（shadcn / Tailwind 等用不了）
  - 混栈（L1 Next.js + L2/L3 Streamlit）= 双倍部署 / 构建 / 样式维护成本

**Alternatives**

1. **保持 Streamlit 全栈**——L1 也用 Streamlit，接受视觉妥协。
2. **混栈**——L1 Next.js + L2/L3 Streamlit。
3. **全栈 Next.js**（采纳）——三层 UI 统一 Next.js 15 + React 19 + TypeScript + Tailwind + shadcn/ui。
4. Vite + React SPA + 独立 FastAPI 后端——两套项目，工程化重。

**Decision**

UI 全栈 Next.js 15（App Router） + React 19 + TypeScript + Tailwind CSS 4 + shadcn/ui（New York style, neutral）+ lucide-react。

**关键变化**：

| 项 | 原 Streamlit | 新 Next.js |
|---|---|---|
| 路由 | 多 Streamlit 进程（不同端口） | App Router file-based |
| 状态 | `st.session_state` | `useState` + Server Actions |
| 长任务 | `st.experimental_rerun` | `useEffect` polling `traces/` |
| 跨页跳转 | URL query param（无中央路由） | Next router + searchParams |
| 后端调 CLI | `subprocess.run`（同进程） | Server Actions (Next.js 14+) 调 subprocess |
| 部署 | 每 console 独立 Streamlit 进程 | 单一 Next.js 进程 + path routing |
| 单页 ≤ 200 行 | Streamlit 单页 ≤ 200 行 | **改为"一屏视觉密度"**——不限行数，组件粒度按需拆 |

**Rationale**

- **一致性 > 行数纪律**：三层一套栈、一份 schema、一份组件库、一个部署。行数只是"一屏看完"的代理指标；换栈后失效，但精神（一屏看完 / 不上 dashboard 组件 / 不展示节点物料给 L1）保留。
- **[ADR-007](007.md) "第一版禁止 framework" 不适用前端**：ADR-007 讲的是 LLM 编排 framework（LangGraph / Temporal 这种状态机抽象）。Next.js 是前端 framework，不在 ADR-007 反对范围。
- **Next.js Server Actions** 让前后端 RPC 类型安全，不需要写 fetch handler。**`s2v` Python CLI 保留不变**，Next.js 只是调 subprocess 的薄壳前端——跟 [ADR-021](021.md) 修正后"git thin 前端"立场一致。
- **作者已是 7 年全栈**——Next.js 学习成本 ≈ 0。"不学新框架"理由对作者不成立。
- **原型 → 实施 路径直**——验收原型用 Next.js，实施仍 Next.js，零重写。

- [`../06-ui-spec.md`](../06-ui-spec.md) §1 纪律 #2（"单页 Streamlit ≤ 200 行" → "一屏视觉密度，组件粒度按需拆"）+ §5 实施栈表 + §6 "多 tab / 多页面"反模式调整（Next.js 路由是 first-class，不是反模式）

新增：

- [`../../ui/`](../../ui/) 项目目录——doc-maker 子项目下的 Next.js 项目，承载 L1/L2/L3 全部 console
- **节点级 UI 目录 ([ADR-019](019.md)) 兼容**：[ADR-019](019.md) 要求 `nodes/<name>/ui/` 装节点 UI。**原型 / 早期实施阶段** L3 集中在 `doc-maker/ui/app/node/[name]/page.tsx`；**后续**可按 ADR-019 拆 L3 到 `nodes/<name>/ui/`（用 Next.js workspace 或 monorepo 拆分）。**MVP 阶段不强制拆**，原型 / 早期实施优先一致性。

不变：

- [ADR-010](010.md) per-node 禁止聚合 —— 仍生效（L2 是 hub 导航，不是聚合）
- [ADR-020](020.md) UI 为作者活体地图 —— 仍生效（L2/L3 立场不变）
- [ADR-021](021.md) git thin 前端 —— Next.js 仍是 git thin 前端（Server Actions 调 CLI）
- [ADR-025](025.md) L1/L2/L3 解耦 —— 实施栈变化不影响立场
- 数据 / 反馈 / 物料仍以 git + 文件系统为 SOT——UI 不存数据库

**反对意见 + 反驳**

| 反对 | 反驳 |
|---|---|
| Next.js 是 framework，违反 ADR-007 | ADR-007 是 LLM 编排 framework，不是前端框架 |
| Streamlit 一页 200 行，Next.js 要 500 行 | 行数翻倍，但配合 shadcn 组件复用可摊销；纪律改为"一屏视觉密度" |
| Web 工程化太重 | 作者 7 年全栈，Next.js 学习成本 ≈ 0 |
| 部署变复杂 | 单 Next.js 进程 < 多 Streamlit 进程（端口管理 / 进程守护） |
| MVP 出口判据（[07-acceptance §5](../07-acceptance.md) #6）要求 "ToyNode Console 上线（Streamlit 单页 ≤ 200 行）" | 同步修正 07-acceptance 第 6 条措辞为"ToyNode Console 上线（一屏视觉密度）" |
