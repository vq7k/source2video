# ADR-001 · 架构形态：Workflow 外壳 + Agentic 节点

**Status**: Accepted

**Context**
- 任务：把**原始语料**（PPT、Markdown 文档、Notion、讲义、视频脚本、研究笔记……任意章节级素材，约 10 分钟讲解量）转成讲解视频。
- 流程骨架可预测：解析 → 内容规划 → 脚本 → 配音 → 画面动效 → 合成。
- 痛点既存在"流程节点多、需要确定性"，也存在"输入方差大、单节点要会判断"。

**Alternatives**
1. 纯 Agent：把原料扔给一个 LLM 自规划。
2. 纯 Workflow：固定 DAG，每节点单 prompt。
3. **Workflow 壳 + Agentic 节点**（采纳）。

**Decision**
外壳走 Workflow（流程骨架写死），内部少数高方差/高失败代价节点（内容规划、脚本、画面决策）允许长成 agentic 能力。

**Rationale**
- 纯 Agent 三大问题：调试地狱、成本失控（token 用量 5–10×）、风格漂移。
- 纯 Workflow 在原料形态多样（图表/文字/代码/动画/中英混杂、跨格式）面前会翻车。
- 真正最优解是"workflow 壳里塞越来越聪明的节点"，而不是把外壳松开。

**Consequences**
- 节点契约（输入/输出 schema）必须先于 agentic 能力到位。
- 何时把一个节点升为 agentic = 后续 ADR 的判断题，不在外壳层决策。
