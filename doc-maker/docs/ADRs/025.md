# ADR-025 · 业务层 Console（L1）· 业务层与框架层 UI 解耦

**Status**: Accepted（2026-05-13）

**Context**
- [ADR-010](010.md) / [ADR-020](020.md) 定下：Per-Node Console（L3）= 作者活体地图，禁止聚合。
- [ADR-021](021.md) 原立场：无 Web UI 上传 source。
- 但 doc-maker 终归要面向"使用者"——可能是作者本人作为"产品用户"，也可能是第二轮的非技术使用者。**这一层 UI 之前文档完全空白**。
- 使用者关心的事：上传素材 → 看 Episode 状态 → 拿产出 → 出问题时跳到框架层排查。不关心节点 / 物料 / Decision Trace。

**Alternatives**
1. 不做业务层 UI——所有人都用 CLI + Per-Node Console。等于把使用门槛拉到作者级别。
2. 业务层 UI 跨越 doc-maker 边界，包含 TTS / Remotion 最终视频。doc-maker 越权。
3. **业务层 UI 在 doc-maker 内部，以 Episode 为单元，跟框架层解耦**（采纳）。

**Decision**

3 层 UI 架构：

| 层 | 名称 | 谁用 | 看什么 |
|---|---|---|---|
| L1 | Business Console | "使用者"（作者本人 / 未来非技术使用者） | Episode 状态、产出、错误入口 |
| L2 | Hub Console | 作者运维视角 | 节点导航 / alive 指示 / 最近 artifact |
| L3 | Per-Node Console | 作者深度排查 | Artifact / Materials / Decision Trace / 反馈 |

**默认隐藏框架层**：L1 跑通的 Episode 不显示 L2/L3 链接；状态 = warn/fail 才显示"→ 进入 X 节点 console"按钮。

**业务层 Console 7 条纪律**：

| # | 纪律 | 违反后果 |
|---|---|---|
| 1 | 以 Episode 为单元，不暴露节点内部 | UI 泄漏框架内部 = L1 立场垮 |
| 2 | 状态 ≥ 详情——状态徽章默认显示，详情折叠 | UI 变运维仪表盘 |
| 3 | 跑通的 Episode 不显示任何节点链接 | "出问题才进入"立场写死 |
| 4 | 不展示 Materials / Decision Trace / Attribution / rubric 分布 | 业务使用者无能力解释，等于噪声 |
| 5 | 核心动作只 3 个：上传 / 重跑 / 接受 | UI 长按钮 = 重新引入运维负担 |
| 6 | 素材上传仅作为 git 前端（内部走 `s2v cases register` + commit） | 绕过 git = 重新造审计/版本化（违反 [ADR-021](021.md) 修正后立场） |
| 7 | 不允许"在 UI 里改 prompt" | 物料改动必经 git + Promote Pipeline（[ADR-008](008.md)） |

**Rationale**
- **不违反 [ADR-010](010.md)（禁止跨节点聚合）**：L1 的单元是 **Episode**，不是 Node。Episode 是业务层概念，跟 node 不是同一个抽象。L1 聚合 Episode 状态 ≠ 跨节点拼大表。
- **不冲突 [ADR-020](020.md)（UI 是作者活体地图）**：那只适用于 L2/L3 框架层 console。L1 是不同立场（使用者视角），并列存在不互相否定。
- **不冲突 [ADR-021](021.md) 修正后立场**：L1 上传素材 = 调 `s2v cases register` 的薄壳，**不绕 git**，全部版本化 / 审计 / frozen 检测仍走 git。
- "跑通的 Episode 不显示节点链接" = 默认隐藏框架层，这是解耦的**实际操作定义**。
- **L1 → L2/L3 跳转单向**：不允许 L2/L3 反向引用 L1 概念（保护框架层不被业务层概念污染——Episode 不进框架内核抽象，[02-architecture.md](../02-architecture.md) §1 / §2 不动）。

**Consequences**

- 修正 [ADR-021](021.md)：扩展 "无 Web UI 上传" → "无 **绕过 git 的** Web UI 上传；允许基于 git 的 thin 前端"。
- [`../06-ui-spec.md`](../06-ui-spec.md) §0 立场加补丁：本文件只讲 L2/L3 框架层 console，L1 见 `_future/business-console.md`。
