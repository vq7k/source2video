# ADR-015 · Train/Holdout 双数据集 + 双向 Gate + Bounded Budget

**Status**: Accepted

**Context**
- "产物不可用怎么办"——用户提出两个常见陷阱：
  1. **大量迭代（蒙特卡洛）**：随机改 prompt 试，碰运气哪次好——重跑就崩。
  2. **过拟合 prompt**：把所有 case 都喂进 prompt，prompt 写得越来越针对当前 case——表面 100%，换新 case 就崩。

**Alternatives**
1. 单 fixtures 集合 + 单向 Gate（经典过拟合）。
2. 单 fixtures + 无 Gate（蒙特卡洛抖动）。
3. **Train/Holdout 70/30 + 双向 Gate + Bounded Budget**（采纳，三件套合一）。

**Decision**
- **Train (70%) / Holdout (30%) frozen 拆分**：fixtures/cases/ 一开始就分。
- **物料迭代期间禁止访问 holdout**：CI 检测 commit 引用 holdout case id 即拒。
- **双向 Gate**：物料 bump 必须 train 改进 +Δ **且** holdout 不退化；任一不满足 CI 拒合并。
- **聚类阈值**：单条反馈不触发改物料；累积 ≥ N 条同 `likely_cause` 才触发（建议 N=3）。
- **Bounded Budget**：同一层物料 ≤ K 次 bump 后 holdout 仍无改进（建议 K=3）→ CLI/CI 禁止继续在该层改，必须升级（见 [ADR-016](016.md)）。

**Rationale**
- 单向 Gate（只看 train）= 经典过拟合陷阱。Holdout 才是过拟合的证伪信号。
- 聚类阈值挡住单条反馈驱动的噪声抖动（蒙特卡洛）。
- Bounded Budget 挡住"死磕 prompt"的完美过拟合——撞 cap 强制升级到下一层。
