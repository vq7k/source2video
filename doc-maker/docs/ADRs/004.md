# ADR-004 · Plan 节点形态：4 步链 + Evaluator-Optimizer

**Status**: Accepted

**Context**
Plan 是"宪法节点"，决定所有 shot 能否独立正确生成。5 轴画像：
- 可验证性：中（schema 可验，内容好坏要 judge）
- 输入方差：高（原料形态千奇百怪：结构化 PPT / 长文档 / 散落笔记 / 视频脚本……）
- 创造性：高（叙事弧、节奏、callback 都是原创判断）
- 失败代价：高（plan 错 → 下游全错）
- 风格敏感度：高（plan 编码所有风格约束）

**Alternatives**
1. 单次 prompt 一发完成。
2. **4 步链 + 自我评估循环**（采纳）。
3. 跳过 plan，让 shot 节点边做边协商（违反 [ADR-003](003.md)，否决）。

**Decision**
内部四步：
1. 内容消化（原料 → 概念图谱）
2. 叙事弧规划（→ hook/setup/.../payoff 结构）
3. Shot 切分（弧 → N 个 shot，含 intent / scope / duration / incoming_context / callbacks / visual_hint）
4. Evaluator-Optimizer 循环：LLM judge 按 rubric 打分 → 不过则修正，**max 3 轮，超出送人工**

**Rationale**
- 四步各自判断维度不同，混一发 LLM 顾此失彼。
- 中间产物可缓存复用（改 target duration 不必重做 Step 1）。
- 每步独立 eval 和迭代，调试空间大。
- 高失败代价节点必须装 self-check 回路。

**Schema 设计三原则**
- `intent` + `scope` 强约束，**措辞不约束**（留 shot 执行空间）。
- `incoming_context` + `outgoing_state` 显式声明（"看意图不看输出"的载体）。
- `style_guide_ref` 用版本引用而非内嵌（系列 const 改了不污染旧 plan）。

**Consequences**
- 节点本身的 eval 必须包含 `executability` 维度（用 plan 切片能否独立产出可用 shot）。
- 不收敛循环：3 轮硬上限是兜底，超出走人工评审队列。
