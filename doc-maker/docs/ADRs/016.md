# ADR-016 · L0–L5 升级路径（撞墙强制升级阶梯）

**Status**: Accepted

**Context**
- [ADR-015](015.md) 的 Bounded Budget 撞墙后，"升级到哪一层"需要明确的阶梯。
- 没阶梯 → 撞墙等于"放弃"；有阶梯 → 撞墙变成"诚实承认这一层不够"。

**Alternatives**
1. 无升级路径（撞墙就停）。
2. **L0–L5 明确升级阶梯**（采纳）。
3. 自动 LLM 决定升哪层（过度自动化）。

**Decision**
代价递增的 6 层：

```
L0  改 prompt 措辞           ← 最便宜
L1  加 / 改 exemplar (few-shot)
L2  改 rubric 维度 / anchor
L3  改 schema（输出结构）
L4  拆节点（任务太大）
L5  重新定义任务            ← 最贵但最诚实
```

撞 Budget Cap → 当前层 CLI/CI 拦死磕，必须升级到 L(n+1)。

**Rationale**
- 死磕 L0 = "完美的过拟合" + 业务永远等不到。
- L5 是诚实的最坏出口——承认 LLM 干不了这事 / 任务定义本身错——比"继续过拟合骗自己"健康。
- 代价递增 = 强制先试便宜的；不让一开始就改 schema 拆节点。
