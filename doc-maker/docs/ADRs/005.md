# ADR-005 · 单节点选型方法论：5 轴画像 + Pattern 阶梯 + 自顶向下降配

**Status**: Accepted

**Decision**
**画像 → 阶梯 → 自顶向下**，三步走。

**Pattern 阶梯**（从下往上爬，够用就停）：
1. 纯代码 — 可验证 + 低方差 + 无创造性。
2. 单次 LLM + 结构化输出 — 中等创造性，schema 可约束。
3. Prompt chaining — 分阶段且每阶段可单独迭代。
4. Evaluator-Optimizer 循环 — 风格敏感 + 可被打分。
5. Tool-using agent — 需要外部信息或多种动作，失败可恢复。
6. Multi-agent — 当前场景排除。

**反直觉做法：先搭最稳最贵跑通，再逐节点降配**。

**Rationale**
- "先省着搭，不行再升"会陷入"模型不够 vs prompt 没写好"的归因黑洞。
- 先证明"任务能做"再优化"用多便宜能做"。
- 每升一层复杂度 ≈ 3–5×，仅在特定节点真正受益。

**Consequences**
- 立项第一周不允许"为了省钱"降档。
- 升级单节点必须有 eval 数据驱动，不靠直觉。
