# learned-rules

> 实战踩坑沉淀（不写理论规则）。

V0.0 起始：0 条。

## 模板

每条 learned-rule 一段：

```markdown
## L<N>: <一句话标题>（<date>）

**事件**：踩到的具体场景（链 commit / session / file 路径）

**规则**：以后怎么避免

**为什么**：根因

**修复模式**（如适用）：具体如何修
```

## 何时写

- 踩了具体的坑（不是理论上的"应该这样"）
- 修复后想到"以后避免这个"
- 跨 session 容易复发的模式

## 何时不写

- 一次性问题（如 typo / 笔误）
- 理论建议（应该 SKILL 化或 ADR 化）
- 业务逻辑细节（应该在 doc-maker 业务文档）
