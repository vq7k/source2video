# ADR-023 · 单仓多子项目 + 框架抽离时机延后

**Status**: Accepted

**Context**
- s2v 项目是基于多个外部独立 repo 的经验**抽象重构**，重新做 LLM-driven 子系统。
- 第一个子项目 = doc-maker（PPT/教材 → script + visual_spec + qa 文档）。
- 未来可能会有 video-maker / tts-maker / publish-maker / analysis-maker 等子项目。
- **复用边界**：astral-pipeline（数据 + 编排）作为外部独立 repo 复用；除此之外**全部在 s2v 里重做**，不复用旧 repo 代码。
- 问题：所有 LLM workflow 子项目共享的"框架内核"（Node Runtime / Materials Registry / Eval Stack / Observability / Feedback Loop / Per-Node Console）该怎么放？

**关键认知（重构期补）**：**不是每个子项目都是 LLM workflow**。

- doc-maker = LLM workflow
- tts-maker（可能）= 纯 API wrapper（ElevenLabs），不需要 framework
- video-maker（可能）= 混合栈（LLM 决策 Python + Remotion 渲染 TypeScript）
- publish-maker / analysis-maker = 几乎不用 LLM

因此 s2v-core 是**条件性共享**：仅当出现**第二个 LLM workflow 子项目**时才有抽离价值。

**Alternatives**
1. 多 repo 各自 vendor 一份——每个 repo 自带完整框架代码。
2. 现在就抽 s2v-core——独立包，所有子项目 `pip install`。
3. **单仓多子项目 + 抽离前提 = 第二个 LLM workflow 子项目开工时**（采纳）。

**Decision**
- s2v 单仓（monorepo 形态）收纳所有 LLM-driven 子项目。
- 当前唯一子项目 = **doc-maker**，物理位置 `s2v/doc-maker/`（代码 + 文档全在子目录内）。
- 仓级 docs/ 极薄，只放仓拓扑（[`../repo-layout.md`](../repo-layout.md)）+ 仓级 ADR（本文件）。
- **抽离前提**：出现**第二个 LLM workflow 子项目**——不是任意第二子项目。
  - 如果第二子项目是纯 API wrapper / 渲染编排 / 数据分析等**非 LLM workflow**，**不触发** s2v-core 抽离，doc-maker 保持自带框架。
  - s2v-core 是 "LLM workflow 子项目的共享框架"，**条件性共享**，不是默认共享。
- 第二子项目身份 = TBD（候选 video-maker / tts-maker / 别的）。
  - TTS 可优先复用旧 harness，不一定 source2video-shaped。

**Rationale**
- **太早抽**（doc-maker 还没跑通就抽 s2v-core）= 抽象不准，API 形态拍脑袋。
- **太晚抽**（两个 LLM workflow 子项目都 drift 半年）= 拉不回。
- **第二个 LLM workflow 子项目开工 = 刚好**：你已经知道哪些是真正稳定的内核 + 子项目还没 drift。
- 跟 [dm ADR-005](../../doc-maker/docs/ADRs/005.md) 阶梯精神一致——"从下往上爬，够用就停，撞痛点再升级"。
- 跟 [dm ADR-007](../../doc-maker/docs/ADRs/007.md) 不冲突：ADR-007 反对"上 LangGraph / Temporal"是外部 framework 绑死抽象。**自己抽自己的包**是另一回事，API 由作者掌握。

**Consequences**
- 当前 `s2v/doc-maker/{src,nodes,fixtures,docs,...}` 物理预留；命名保持业务无关（不用 `doc_maker_runtime`，用 `runtime/`），方便未来抽离。
- 跨仓边界（pipeline_io）走 astral-pipeline `episodes/<id>/raw/`——[dm ADR-021](../../doc-maker/docs/ADRs/021.md) 精神延伸。
- 第二个 LLM workflow 子项目开工 = **触发一次集中重构**：doc-maker/src/ → s2v-core/。
- 子项目技术异质允许：每个子项目自己定技术栈，仓级只共享 git / CI / 跨仓接力契约。
