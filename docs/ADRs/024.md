# ADR-024 · 整体架构 = DAG（4 层），演化层循环不破坏 DAG 性质

**Status**: Accepted

**Context**
- 子项目专属 ADR 没有正面声明"整体架构 = DAG"——只是从 [dm ADR-003](../../doc-maker/docs/ADRs/003.md)（shot 不互读）/ 不变量 #7（Node 通过 plan 中介）/ #9（Node 不读其它 Artifact）隐式推导出来。
- 未来"复杂编排"撞墙时（如 Phase 3 跨节点反馈 / 条件路由 / sub-workflow）需要明确判据——**哪条 DAG 性质被破坏了**。
- 演化层（反馈→改物料→bump→重跑）是循环——容易跟"执行层是 DAG"混淆。

**Decision**

声明 **s2v 架构是 4 层有向无环图**：

```
L1  s2v 仓
     │ 包含
L2  子项目         doc-maker → tts-maker → video-maker          DAG ✓
     │             （通过 astral-pipeline 接力，不直接互调）       顺序依赖
     │ 包含
L3  子项目内的节点  plan → shot×N → qa                          DAG ✓
     │             （ADR-003 不互读，plan 中介）                   树状 fanout
     │ 包含
L4  节点内的步骤    Step 1 → Step 2 → Step 3                    DAG ✓
                   (含并行：Step 2 ∥ Step 3 都依赖 Step 1)         含并行分支
```

**两层语义**：

| 层面 | 性质 | 周期 |
|---|---|---|
| **执行层**（单次跑批） | 严格 DAG | 一次 run |
| **演化层**（多次跑批之间） | 循环（反馈→改物料→重跑） | 跨 N 次 run，每次都是新 DAG instance |

**演化层循环不破坏 DAG 性质**——它是"DAG 实例之间的关系"，不是"DAG 内部的边"。

**Alternatives**
1. 不显式声明——从 [dm ADR-003](../../doc-maker/docs/ADRs/003.md) / 不变量隐式推导，新人不易抓住整体性质。
2. **显式 ADR 声明 4 层 DAG**（采纳）。
3. 把"演化层"也画进图——破坏 DAG 简洁性。

**Rationale**
- DAG 不是装饰术语，**决定 5 件事**：并行化 / 缓存 / 失败定位 / 重试粒度 / 可视化。
- 显式 ADR 化 = 给新业务 / 新子项目一个 30 秒理解架构的入口。
- DAG 性质破坏的早期警报：任何"跨 Node 横向边" / "执行层循环" 都是信号，立刻回到 [dm ADR-003](../../doc-maker/docs/ADRs/003.md) / 不变量 #7 #9 审视。

**DAG 性质在当前架构里隐式落到哪儿**（dm ADR 引用见 [`../../doc-maker/docs/ADRs/`](../../doc-maker/docs/ADRs/)）：

| 设计 / 不变量 | 它依赖 DAG 性质 |
|---|---|
| 不变量 #9 Node 不互读 Artifact | DAG 边只能 "上游→下游"，禁止横向边 |
| 不变量 #7 Node 通过 plan 中介 | DAG 节点间通过显式中间产物连接 |
| [dm ADR-003](../../doc-maker/docs/ADRs/003.md) shot 之间不横向通信 | shot 在 DAG 里是兄弟节点，互不连边 |
| 中间产物缓存（[`../../doc-maker/docs/02-architecture.md`](../../doc-maker/docs/02-architecture.md) §2.1） | DAG 节点 input hash 命中即跳过 |
| asyncio 并发（[`../../doc-maker/docs/02-architecture.md`](../../doc-maker/docs/02-architecture.md) §2.1） | DAG 拓扑序并行执行 |
| Node 独立 retry / 失败保留 | DAG 节点是 retry / 重做的最小单位 |
| Eval-Opt max N 轮 | 单 Node 内部 retry 不是图循环——从外看仍 input→output |
| Bounded Budget K 轮 | 物料演化层循环，不在单次跑批 DAG 里 |

**DAG 破坏的早期警报信号**：

| 信号 | 立刻回 |
|---|---|
| "节点 A 想读节点 B 的输出（不经过 plan）" | [dm ADR-003](../../doc-maker/docs/ADRs/003.md) / 不变量 #9——这是横向边 |
| "执行流出现循环 / goto" | 不变量 #7——违反 DAG |
| "状态机跨节点共享" | [dm ADR-007](../../doc-maker/docs/ADRs/007.md) 升级触发条件——可能要上 LangGraph，但先确认不是误判 |

**Consequences**
- 不变量 #9 / #7 / [dm ADR-003](../../doc-maker/docs/ADRs/003.md) 升级为"DAG 性质的具体落点"，引用本 ADR。
- 跨节点反馈 / 条件路由 / sub-workflow 等未来需求 = **DAG 升级触发条件**——届时立 ADR 显式说明哪条性质被改变。
- mermaid / graphviz 可作为标准的可视化形式——文档新增图都按 DAG 风格画。
