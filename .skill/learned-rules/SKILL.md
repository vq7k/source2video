# skill: learned-rules

追加一条踩坑规则到 `.agents/learned-rules.md`。

## 何时用

**只在实战踩坑后**追加。**不写理论规则 / 行业最佳实践 / 我觉得应该这样**。

## 一条规则的格式

```markdown
### <规则一句话>

- **场景**：什么时候触发
- **踩坑**：具体踩到什么（数据 / 现象）
- **规则**：应该怎么做
- **来源**：commit hash / session id / 文档路径
```

## 步骤

1. 实战踩到坑后立刻写（不要"我以后补"）
2. 打开 `.agents/learned-rules.md`
3. 按格式追加
4. session 结束前 commit

## 反例

- ❌ "三振出局：3 次未解升级" — 这是理论
- ❌ "应该用 TypeScript 严格模式" — 这是规范不是踩坑
- ❌ "今天累了 commit 一下" — 这不是规则

## 校验

每条规则必须含 4 段（场景 / 踩坑 / 规则 / 来源）。无来源的不算规则。

## 何时拆出去

如果某条规则可以**编码化**（脚本 / hook / 自动检查），把它实现成代码，再从 `learned-rules.md` 删除。规则文件不养永久依赖。
